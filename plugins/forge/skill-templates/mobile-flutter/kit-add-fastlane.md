## When to use

- Triggers: "set up Fastlane", "add CI deploy lanes", "wire App Store / Play Store upload", `/kit-add-fastlane`.
- The user wants automated build + distribution lanes for this Flutter project.
- Prerequisites: Fastlane must be installed (`gem install fastlane` or via Bundler). Apple Developer + App Store Connect access required for iOS lanes. Google Play service account JSON required for Android lanes.

## What this skill produces (FORGE-6 target state)

- `fastlane/Appfile` — app identifiers per flavor.
- `fastlane/Fastfile` — lanes: `beta` (TestFlight / Firebase App Distribution), `deploy_prod` (App Store / Play Store), `sync_certs` (match / cert + sigh).
- `fastlane/Matchfile` — certificate storage config (git-based or S3).
- `.env.fastlane` (gitignored) — API key paths, team IDs, service account JSON path.
- CI workflow (GitHub Actions or similar) that calls the lanes on tag push.

## Pre-conditions (complete BEFORE running lanes)

1. **Launcher icons** — replace the placeholder PNGs at `assets/icons/launcher_<flavor>.png` with real artwork (1024×1024 recommended). Then run:
   ```sh
   dart run flutter_launcher_icons -f flutter_launcher_icons-dev.yaml
   dart run flutter_launcher_icons -f flutter_launcher_icons-prod.yaml
   # repeat for each flavor
   ```
2. **Apple Developer** — App ID registered, bundle ID matches `Q1_3_bundle_id` in each flavor (`<bundle_id>.dev`, `<bundle_id>.prod`, etc.).
3. **App Store Connect API key** — JSON key file downloaded; path stored in `.env.fastlane` as `APP_STORE_CONNECT_API_KEY_PATH`.
4. **Google Play service account** — JSON key with "Release Manager" role; path stored in `.env.fastlane` as `GOOGLE_PLAY_JSON_KEY_PATH`.
5. **match repo** — private Git repo (or S3 bucket) for certificate storage.

## Planned lanes (FORGE-6 implementation)

### `beta` lane

Builds a signed release IPA / AAB and uploads to TestFlight (iOS) + Firebase App Distribution or Play Internal track (Android).

```ruby
# fastlane/Fastfile (planned — FORGE-6)
lane :beta do |options|
  flavor = options[:flavor] || "dev"
  # iOS
  sync_code_signing(type: "appstore")
  build_ios_app(
    scheme: flavor.capitalize,
    export_method: "app-store",
    output_directory: "build/ios/",
  )
  upload_to_testflight(skip_waiting_for_build_processing: true)
  # Android
  gradle(task: "bundle", flavor: flavor, build_type: "Release")
  upload_to_play_store(track: "internal", aab: "build/app/outputs/bundle/#{flavor}Release/app-#{flavor}-release.aab")
end
```

### `deploy_prod` lane

Promotes an approved TestFlight build to App Store / Play production.

### `sync_certs` lane

Runs `match` to sync or refresh signing certificates and provisioning profiles.

## Steps (when FORGE-6 lands)

1. `bundle init` + `echo 'gem "fastlane"' >> Gemfile` + `bundle install`.
2. `fastlane init` — follow prompts to generate `Appfile`.
3. Write `Fastfile` with lanes above.
4. `fastlane match init` — configure certificate storage.
5. `fastlane sync_certs` — fetch/create certs + profiles.
6. `fastlane beta flavor:dev` — first beta build smoke test.
7. Wire lanes into CI (GitHub Actions `.github/workflows/deploy.yml`).
