## Edge calls

- **`skills/` — IN**: Two subdirectories (`peon-ping-config/`, `peon-ping-toggle/`), each containing only a `SKILL.md`. These are manually authored skills not managed via the plugin store. They must be versioned alongside `commands/` and `agents/`. Decision: IN.

- **`plugins/` — OUT**: Plugin store manages these; `installed_plugins.json` (also inside `plugins/`) lists 8 plugins with install paths. On a new machine, `/plugin install` re-fetches everything. Gitleaks detected false positives in `plugins/cache/` (Figma plugin docs use example Figma fileKeys like `kL9xQn2VwM8pYrTb4ZcHjF` that pattern-match as generic-api-key). These are docs examples, not real secrets — confirmed by context. OUT regardless since plugins/ is re-fetched.

- **`backups/` — OUT**: Contains `.claude.json.backup.*` files. Gitleaks detected a `X-Goog-Api-Key` in at least one backup. Likely an MCP session key captured at backup time. OUT and gitignored.

- **`ide/` — OUT**: Contains `61054.lock` with an `authToken` field. IDE session authentication token, per-machine. OUT.

- **`keybindings.json` — ABSENT**: File does not exist on this machine. No action needed; if created in future it would be IN (cross-machine UX consistency).
