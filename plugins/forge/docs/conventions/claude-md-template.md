# CLAUDE.md template

Skeleton CLAUDE.md generated by `/project-init`. Stack-specific lines marked `<!-- STACK -->`.

The template uses **references**, not copies, for global rules. Project-specific deviations override globally-referenced sections via the project's own `docs/` folder.

## Per-stack populate examples

### mobile-flutter

```
Run dev          | flutter run --flavor dev -t lib/main_dev.dart
Run prod         | flutter run --flavor prod -t lib/main_prod.dart
Format           | dart format .
Lint             | flutter analyze --fatal-warnings
Test             | flutter test
Test integration | flutter test integration_test/
Codegen          | dart run build_runner build --delete-conflicting-outputs
L10n             | flutter gen-l10n
Build            | flutter build apk --flavor <flavor> -t lib/main_<flavor>.dart
```

## How `/project-init` uses this

1. Asks user about stack, hosting, DB, framework
2. Reads this skeleton
3. Substitutes `<!-- STACK -->` markers with stack-specific lines (per "Per-stack populate examples")
4. Asks user for project-specific Mandatory rules (or proposes defaults from skill-templates)
5. Writes the result to `<project>/CLAUDE.md`
6. Initializes `<project>/docs/00_meta/decisions-log.md` with a first entry pointing to the global rules
7. Copies relevant kit-* templates from `~/.claude/skill-templates/<stack>/` to `<project>/.claude/skills/`
