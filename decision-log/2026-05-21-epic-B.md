## [orchestration] Subagent-per-ticket vs inline execution

**Context:** EPIC B has 7 sub-issues, all SKILL.md rewrites averaging 100-150 lines each. Total context cost if inline: ~50K tokens reading + editing.

**Problem:** Context economy is the binding constraint for the rest of the overnight run (still need C+D, G, H + final README after this).

**Chosen:** Dispatch one subagent per ticket using `forge:execute-ticket`-style curated prompts. Main session only orchestrates: branch + version bump + dispatch + commits + epic-close.

**Why:** Subagent context is ephemeral and doesn't pollute the main thread. Trade-off: subagents may make slightly different design choices than I would — acceptable, all skills already conform to `forge:writing-skill` rule-sets via `audit.sh`.
