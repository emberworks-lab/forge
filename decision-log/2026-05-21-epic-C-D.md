**Context:** `plugins/forge/skill-templates/` already contains `web-react` and `backend-node`. Tickets ask for `web-nextjs` and `backend-nest`.

**Problem:** Are `web-react`/`backend-node` outdated versions, parallel templates, or did the tickets pick names without checking?

**Options considered:**
- **A:** Add `web-nextjs/` and `backend-nest/` as new directories — keeps current templates intact, allows future deletion
- **B:** Rename `web-react/` → `web-nextjs/`, `backend-node/` → `backend-nest/` (also requires updating any consumers)
- **C:** Inspect existing templates first; rename if they're roughly the right shape, add new if substantially different

**Chosen:** **C — let the subagent inspect and decide**
**Why:** Either rename or add-new is fine; the subagent doing the extract is best placed to judge after seeing the current content. I'll instruct subagents to prefer rename if the existing template is the right base, add-new otherwise.
