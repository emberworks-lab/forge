## Proposal: write skill tests (worth doing — user asked)

superpowers (`github.com/obra/superpowers/tree/main/tests`) has two test tiers we should copy:

1. **skill-triggering tests** (`tests/skill-triggering/`): a naive prompt file per skill → `claude -p "<prompt>" --plugin-dir <plugin> --dangerously-skip-permissions --max-turns 3 --output-format stream-json` → grep the JSON log for `"name":"Skill"` + `"skill":"<name>"`. Asserts a skill TRIGGERS on a natural prompt without being named. Cheap to author (one prompt file + the shared `run-test.sh`), high value for catching description/routing regressions.
2. **fixture/behavioral tests** (`tests/subagent-driven-dev/go-fractals`, `svelte-todo`): a small fixture repo where the skill runs a real task end-to-end and the outcome is asserted. Heavier (model-in-loop) but the only way to catch real behavioral breakage.

**Recommended forge plan (new epic, e.g. "EPIC J — skill test harness"):**
- `plugins/forge/tests/structural/` — wrap the existing `audit.sh` as the deterministic, no-model CI tier (already exists; just formalize a `run-all.sh`).
- `plugins/forge/tests/triggering/` — port superpowers' `run-test.sh` + naive prompts for the auto-invoked skills first: `epic-close`, `execute-ticket`, `e2e` routing, `commit`, `diagnose`, `update-docs`.
- `plugins/forge/tests/fixtures/` — start with ONE fixture that doubles as #62: a scratch repo where `/forge:project-init` runs and asserted scaffold files appear; then `forge:simplify` on a known-duplicate file; `forge:e2e --init` scaffolding.
- This would convert several "human-pilot" deferrals (#62, parts of #77) into automated checks.

Caveat: triggering + fixture tests need `claude -p` headless runs (billable, slow) — keep them out of the inner loop; run in CI / on demand. `audit.sh` stays the fast gate.

## Gotchas for the next agent

- **`Closes #a, Closes #b`** — repeat the keyword per issue, or auto-close silently drops all but the first (the bug above).
- **Parallel subagents on the SAME working tree race on the git index** — if dispatching parallel skill-authoring subagents, have them write files but NOT commit; commit yourself sequentially. (Two commits got mis-attributed this session; squash-merge hid it, but don't rely on that.)
- **`audit.sh` check 6** forbids `superpowers:*` in any SKILL.md body. Check 7/10: any "subagent"/spawn mention must name `sonnet`/`opus`; never `haiku` outside backticks.
- **Every SKILL.md must pass audit** before commit: `bash plugins/forge/skills/writing-skill/scripts/audit.sh <path>`.
- **Don't hand-write HTML docs** (format law). Markdown source only; HTML is generated.
- A pre-commit hook runs `code-review-graph` (prints INFO lines) — harmless.
